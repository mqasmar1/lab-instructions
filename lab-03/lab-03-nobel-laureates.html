Understanding Merge Conflicts

As you work with Git and GitHub, you may occasionally encounter merge conflicts. While less common when working independently, it’s important to understand what they are.

What causes a merge conflict?

A merge conflict occurs when:

  • You make changes to your local files
  • Someone else (or you on another computer) makes different changes to the same part of the same file on GitHub
  • Git can’t automatically determine which changes to keep

What does a merge conflict look like?

Git puts conflict markers in your code:

<<<<<<< HEAD 
Your changes are here
======= 
Their changes are here
>>>>>>> hash123abc

How to resolve a merge conflict:

Step 1. Edit the file to keep the changes you want

Step 2. Delete the <<<, ===, and >>> markers

step 3. Save the file

Step 4. Stage, commit, and push

Best practices to avoid merge conflicts:

  • Always pull before you start working
  • Commit and push frequently
  • If you do get a conflict, resolve it immediately before continuing work
  • When in doubt, ask for help!

Most Living Nobel Laureates Were Based in the US When They Won Their Prizes

… says the Buzzfeed article. Let’s see if that’s true.

First, we’ll create a new variable to identify whether the laureate was in the US when they won their prize. We’ll use the mutate() function for this. The following pipeline mutates the nobel_living data frame by adding a new variable called country_us. We use an if statement to create this variable.

The if_else() function tests a condition. If TRUE, it returns the first value (“USA”); if FALSE, it returns the second value (“Other”).

nobel_living <- nobel_living %>%
  mutate(
    country_us = if_else(country == "USA", "USA", "Other")
  )

What this code does:

  • If country equals "USA", set country_us to "USA"
  • Otherwise, set country_us to "Other"

Next, we will limit our analysis to only the following categories: Physics, Medicine, Chemistry, and Economics.

nobel_living_science  <- nobel_living %>%
  filter(category %in% c("Physics", "Medicine", "Chemistry", "Economics"))

Common Errors and Troubleshooting

Data wrangling errors:

  • Error: “object ‘nobel_living’ not found” → Make sure you ran the code chunk that creates this data frame
  • Error: “object ‘nobel_living_science’ not found” → Make sure you ran the code chunk that creates this filtered data frame
  • Filter not working as expected → Check your logic: == for equals, != for not equals, is.na() for missing values
  • if_else() not working → Make sure the condition comes first, then the TRUE value, then the FALSE value

Visualization errors:

  • Bars not showing correctly → Check that you’re using the right variable for x-axis and fill
  • Facets not appearing → Make sure you’re using facet_wrap(~category) with the tilde
  • Bars are vertical instead of horizontal → Add coord_flip() to your plot
  • Colors not showing segments → Use fill = born_country_us inside aes(), not color

General issues:

  • Code chunk not running → Make sure you changed eval=FALSE to eval=TRUE
  • Inline code not working → Use backticks with r
  • Changes not on GitHub → Remember to commit AND push
